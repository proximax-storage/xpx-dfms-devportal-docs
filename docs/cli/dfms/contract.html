dfms contract compose

This command composes Contract for new Drive between current DFMS node and multiple DFMSR nodes with conditions to store space amount of data for subscription time period.

First of all, a Contract invitation is built from configured arguments and options. This invitation broadcasted throughout the network. Then, replicator receive the invitation and decide to accept it or not. Their decision is based on the configured fields. The composing, at the same moment, awaits the configured amount of replicator to accept invitations. If throughout 10 minutes the minimal amount of replicator do not accept the invitation - composing fails. In another case, it succeeds. From that moment new Drive is considered to be created.

Arguments

NameDescription
spacerequired space (in MB)
durationDrive duration (in month)

Options

NameDescription
replicas(r)to set the amount of Drive replicas. One replica = one replicator. The default is 3
min-replicators(mr)to set a minimum amount of joined replicator for composing to succeed. The default is 3
subscription-price(p)to set a price in tokens for one subscription period. The default is space * replicas
number-subscription-periods(sn)to set number of desired subscriptions. The default is 3
percent-approvers(pa)to set percent of replicator needed to come up to consensus to proceed any Drive account related action
private-key(key)to set a custom private key to generate Drive account from
helpto see description with examples

Examples

  • Create a new contract with the needed parameters:

    $ dfms contract compose 1000 1 \        # size & duration
                --replicas=5 \
                --min-replicators=3 \
                --billing-price=5000 \
                --billing-period=1 \  
                --percent-approvers=67
